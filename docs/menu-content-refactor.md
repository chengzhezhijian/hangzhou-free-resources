# 菜单页内容重构（v69）

## 背景

场景 / 工具 / 说明 / 反馈等子页文案仍引用旧规模（如 2993 条、72 城），与首页实际数据（13000+ 条、107 城）脱节。

## 改动摘要

| 菜单 | 重构前 | 重构后 |
|------|--------|--------|
| **场景** | 静态 9 卡 +「72 城均可查」 | 首页 6 个场景 chip 带动态计数；每卡显示「X 处可查」；描述含实际城数 / 杭州细录 |
| **工具** | 仅外链官方平台 | 新增「站内能力」4 卡（地图导航、距离排序、城市筛选、场景/设施）；外链按全国/省/市分组 |
| **说明** | 硬编码 2993 条、400+ 杭州 | JS 从 `RESOURCES` 注入设施亮点、覆盖范围、使用指南、数据说明 |
| **反馈** | 单一问卷按钮；底部 nav 图标不一致 | 双渠道卡片（问卷 + 页内表单）；`<dialog>` Sheet 与详情叠层一致；nav 图标统一 |
| **全站 tagline** | `72城搜完即走` 写死 | `site-config` 模板 `{cities}` + `SiteStats` 动态替换 |

## 新增文件

- `js/site-stats.js` — 从 `RESOURCES` / `QUICK_SCENES` / `SCENE_GUIDES` 派生统计，供子页与 layout 共用

## 修改文件

- `js/pages.js` — 场景 / 工具 / 说明 / 反馈动态渲染
- `js/layout.js` — tagline、反馈渠道按钮
- `js/site-config.js` — tagline 模板
- `js/data.js` — 移除场景指南中「72 城」硬编码
- `guide.html` / `tools.html` / `about.html` / `feedback.html` — 结构占位 + v69 样式栈
- `index.html` — v69、引入 site-stats、meta 去 72 城
- `css/style.css` — 场景 chip、计数徽章、反馈渠道、站内工具卡
- `scripts/test-ui-quality.mjs` — 菜单 DOM / 无过时数字 / 动态模块

## 验收步骤

1. `npm run test:all` 全绿
2. 打开 **场景**：顶部 chip 有遛娃/自习等 + 数量；卡片右上角有「X 处可查」
3. 打开 **工具**：首段「站内能力」4 卡；下方官方链接可点击
4. 打开 **说明**：数据说明含当前总条数（非 2993）；footer 有日期与规模
5. 打开 **反馈**：问卷 + 页内表单两渠道；点表单弹出 Sheet
6. 全站 header 副标题显示「107城搜完即走」（随数据变化）
